# 到底要不要考研

## 前言 {{id: 'info'}}

本文作者为北大毕业老研究生一枚。写这篇文章的目的既不是劝大家读研非常好，速来，也不是专门写一个帖子来劝退大家，而是希望各位在考研与找工作之间做出选择时有一份参考清单。

因此，本文适合的对象包括:

1. 想随大流的
2. 没有明确目标的
3. 选择了其一, 但觉得另一项可能会更好的

如果你当前的阶段属于以上三种类型，希望本文对于你有所帮助。同样地，对于以下两种情形，本篇文章或许不那么适合:

1. 已经做出选择，并明确知道自己为什么要这么选的
2. "我妈让我" / "我家里让我"的

第一种情形是最理想的状态。如果你是第一种情形，那么恭喜你，懂得规划自己的生涯是一件非常好的事，并且也非常明确自己想要什么。事实上，**这种情形应是大多数人的理想状态**。

第二种情形的，低情商的说法是先去治好妈宝病，高情商的说法是相比起家人的期望，还需要回答一个更重要的问题：**你自己对你自己的期望是什么**? 只有这个问题想明白了，讨论生涯规划的问题才有意义。

## 考研前你需要考虑的事 {{ id: 'before-post' }}

下面是第一阶段问题清单，照着下面的问题给出自己的答案:

1. 当前阶段下，于你自己而言，读研和找工作各自的优劣是什么?
2. 身边的人为什么读研?
3. 你自己为什么读研?
4. 如果前三个问题最终导向都是"读研", 那么你想好要读哪所学校的什么专业了吗？

可以看出，上面四个问题最终都可以归结为两个大问题：**"是什么"**和**"为什么"**，至于**怎么做**，必须建立在上面四个问题的基础上才能继续探讨。下面，我们分问题拆解:

> 问题一: 当前阶段下，于你自己而言，读研和找工作各自的优劣是什么?

这是四个问题中的第一个问题。其实答到最后，你的答案只有如下二者之一, 不会有中间选项:

1. 找工作比读研更有优势
2. 读研比找工作更有优势

如果最终的结论是答案一，那么剩下三个问题可以自动跳过了。如果是答案二，想必至少问题二和问题三你会有答案了。那么，怎么去得出最终结论呢？有这么几个子问题需要考虑:

1. 你要走学术道路吗?
2. 你对未来工作的期望是什么? 
3. 当前阶段的资源、能力，包括但不限于代码水平、实习经历、学历好坏等，可以支撑你找到符合预期的工作么？

上述问题1是问题2的子问题。但为什么我要单独拿出来问一遍呢? 一是因为学术道路并不好走，如果你并非天赋非常好，只是感觉发论文很厉害我也要发，那么我觉得还是慎重考虑。更直白地讲，如果你**本科院校不是211，那么你根本不要想走学术道路**，因为几乎没有可能走的很长远。二是因为，如果**笃定了学术道路走到黑，你只有读研+读博一条路可走**，只需要考虑去哪读+怎么读的问题。

所以，下面我直接从问题2谈起。就以互联网大厂为例，大厂岗位会有很多种：技术岗（前端(包括Android / iOS)、后端、算法），产品经理，运营，不一而足。如果你单纯就是想进互联网大厂，那么你要想好的一点是，你是想干技术，还是想做产品，还是想去职能部门。大厂工作体验日后开坑谈，当前阶段只解决对于“想要什么”的启发性问题。

就目前我所知道的，如果想做技术, 有如下两点参考:

1. 必须承认的是，算法岗是技术薪资天花板，**前后端怎么卷也不会比算法岗的工资高**。但同时也必须承认的是，算法岗现在的门槛已经非常高了。就目前而言，说句直白的，**如果你的本科院校不是211以上，或者不是像杭电一类的计算机强校，那么你够到算法岗的门槛就是博士了**。另一个事实是，**算法岗的学历卷度，前列985硕士只是刚刚碰到了门槛，大厂根本不缺C9和清北硕博的人来做算法，甚至博士都已遍地走了**。如果你就是想干算法岗，非算法不可，那么你不仅要读研，还要读强校强导师的研，甚至如果能读博更好。
2. 如果做前后端，那么我觉得必要性并不是很高，但也有需要读研的情况。我会在第三个问题的拆解中分析。

如果想做PM，那么我想说的是今昔已不同于往日，PM岗的学历卷度也起来了，**想拿大厂PM职位已经硕士或名校本科起步了**。

如果只想做诸如HR, 品牌宣传等**职能岗，那么读研的必要性基本为0**.

上面的分析是针对问题2的回答。总之，你要考虑的是自己想做的职位一你能不能做，二吃不吃学历饭。问题二想清楚后，自然问题三也迎刃而解。还是拿上面的例子来说:

**必须读研的情况**:

1. 只想做算法岗，非算法不去。如果**本科学历并不是很好，那么只剩下读名校硕博一条路可走，而且必须读到博才会有机会**。如果基础学历就很好，硕士毕业后就已经够到门槛了。
2. 本科非计算机/软工，但就是喜欢写代码，想干这一行，读研跨考还是有必要的。
3. 对写代码没兴趣，但还是想进大厂，干PM也行。

**慎重考虑的情况**:

1. 岗位并不需要学历很好，但需要的是你的代码能力、做事能力，比如前后端。如果你本科期间就已积攒了足够的能力，或者说实习履历刷的不难看，那么读研只个可选项，甚至根本没有任何必要读。
2. 干职能岗，但要接受职能岗的薪资是大厂里比较低那一档的事实。

问题一考虑清楚后，你或许已不需要考虑接下来的问题了，或者你可以进入接下来的问题考虑了。

> 问题二: 身边的人为什么读研? && 问题三: 你自己为什么读研?

这两个问题可以合并在一起考量。问题二并不需要你知道每一位考研的同学背后的故事，而是为问题三做铺垫，问题三才是对你来说至关重要的。

首先还是谈论问题二。身边人考研的目的，有的可能是为了追求名校情结（比如作者本人），有的可能是为了各种各样的原因。关心这个问题是因为如果身边考研氛围浓厚，你非常容易掉进随大流的陷阱里。比如你已经回答了问题一，你觉得没必要考，结果身边同学一个个都在备考让你觉得你也得考一个，这种情况还是不要出现的吧。

问题二讨论清楚之后我们再来讨论问题三。如果你在问题二上不断打转，觉得别人考了我也得考一个，那么我的建议是退回问题一重新考虑并直接进入此问题。如果你能明确知道别人考是为了更好的工作、为了转行等，那么你也要给出自己的答案了。原因不外乎如下几种:

1. 名校情节。这是一个原因，但你自己必须考虑清楚对于你来说是不是重要的
2. 未来规划需要有一个研究生学历
3. 其他原因

除了随大流以外，任何原因都是合情合理的，只要你为你的选择足够负责即可。

> 问题四: 如果前三个问题最终导向都是"读研", 那么你想好要读哪所学校的什么专业了吗？

这个也挺重要的，但第一阶段可以只回答两个问题:

1. 本专业还是跨专业?
2. 本校还是外校?

由于本文并不是专注于特定院校、特定专业分析的，而这一部分建议直接看张雪峰视频就行，所以本文提供一个大致可参考的角度:

1. 本专业本校最容易考上
2. 跨考的话，跨文科比跨理科简单，跨有统考的比没统考的简单
3. 外校的话，学校段位越高难度越大

在选择前，按照这三个维度分析，答案基本上已经有五六成把握了。剩下的三四成，既可以发邮件(896175698@qq.com)向我咨询, 也可以找张雪峰的讲座、视频等获得启发。

## 结语 {{ id: 'conclusion' }}

在正式决定考研之前，能把第一阶段的四个问题都想明白，才可以进入第二阶段，也就是“怎么做”的阶段。本文提出了第一阶段的四个需要考虑的问题，并对逐个问题拆解了一番。如果还在读与不读的这个问题上犹豫不决，那么我认为就拿这四个问题当一个checklist去考虑还是应该的。希望能对有缘读到这篇文章的你有所启发。

第二阶段问题及拆解，也就是“怎么做”，占坑待填。
