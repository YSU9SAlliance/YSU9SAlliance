D.二叉树：
二叉树是一种常见的树形数据结构，它由若干个节点组成，每个节点最多有两个子节点，分别称为左子节点和右子节点。二叉树具有以下特点：

1. 每个节点最多有两个子节点，分别称为左子节点和右子节点。如果某个节点没有子节点，则称其为叶子节点。

2. 二叉树的子节点顺序是有序的，即左子节点在前，右子节点在后。

3. 二叉树的每个节点都包含一个值，称为节点值。

4. 二叉树通常用于表示分层数据结构，例如树形目录、组织结构等。

5. 二叉树的深度由根节点到最深叶子节点的距离决定，深度为树中节点的最大层数。

6. 二叉树可以是空树，即不包含任何节点的情况。

根据二叉树的性质，可以将二叉树分为多种类型，例如满二叉树、完全二叉树、平衡二叉树等。二叉树在计算机科学中有着广泛的应用，常用于实现各种算法和数据结构，例如二叉搜索树、堆、表达式树等。
二叉树中大多使用递归求解
E.二分查找：
1. 初始化左右边界：将查找范围确定在数组的起始和结束位置。
2. 计算中间位置：取左右边界的中间位置作为当前查找范围的中点。
3. 比较中间值：将目标值与中间位置的值进行比较。
4. 调整边界：根据比较结果，确定目标值可能在哪一部分，然后调整左右边界。
5. 重复步骤2到步骤4，直到找到目标值或者确定目标值不在数组中。

G.贪心算法：
贪心算法（Greedy Algorithm）是一种在每一步选择中都采取在当前状态下最优的选择，从而希望最终能够达到全局最优解的算法。在每一步选择中，贪心算法都会做出当前状态下最优的选择，而不考虑该选择对之后的选择造成的影响。贪心算法的核心思想是局部最优解能够导致全局最优解。

贪心算法的特点包括：

贪心选择性质：即每一步都选择当前状态下最优的解，而不考虑之后的决策会如何影响全局最优解。

最优子结构性质：问题的最优解可以通过子问题的最优解来构造。

无后效性：即某个状态以后的过程不会影响以前已经做过的决策。

贪心算法适用于一些特定类型的问题，例如最小生成树、最短路径、区间覆盖等问题。然而，并非所有问题都适合贪心算法，因为贪心算法的局部最优选择并不一定能够导致全局最优解，有些问题可能需要使用动态规划等其他方法来求解。
买卖股票的最佳时期：只考虑了当前的利润情况，而没有考虑未来可能的价格变化从而实现贪心算法
H.动态规划：
动态规划（Dynamic Programming，简称 DP）是一种解决多阶段决策过程最优化问题的数学方法，也是一种算法设计技术。它将问题分解成若干个子问题，并按照一定的规则将子问题的解组合起来，从而得到原问题的解。动态规划通常用于解决具有重叠子问题和最优子结构性质的问题。

动态规划的基本思想可以归纳为以下几个步骤：

1. 划分子问题：将原问题划分成若干个子问题，这些子问题应该具有重叠性质，即原问题的解可以通过子问题的解组合而来。

2. 确定状态：确定问题的状态，每个状态表示问题的一个子集，动态规划的状态应该满足无后效性，即某个状态以后的过程不会影响之前已经做出的决策。

3. 确定状态转移方程：建立状态之间的转移关系，描述子问题之间的联系，并且利用子问题的解来构造原问题的解。

4. 寻找边界条件：确定初始状态和边界条件，即问题的最简单情况。

5. 自底向上求解：通过填表格或者利用递归等方法自底向上地求解问题，直到求解出原问题的解。

动态规划常用于求解最优化问题，例如最短路径、最大子序列和、背包问题等。它能够通过有效地利用子问题的解来减少重复计算，从而提高算法的效率。

# 2.面向对象思想总结：
面向对象编程，也叫OOP，是相对c语言的一种新的编程范式，模拟了现实世界的事物，将事物看作由属性，行为（和子对象）构成的对象，由类实例化得到。类可看作图纸。
函数重载的特点包括：
1. 函数名称相同：重载的函数必须具有相同的名称。
2. 参数列表不同：重载的函数必须具有不同的参数列表，包括参数的数量、类型或顺序。返回类型不同不能用来区分重载。
3. 相同作用域：重载的函数必须在同一个作用域内。
